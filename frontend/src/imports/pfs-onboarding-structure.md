PFS Onboarding Structure
Principles:
Simple. Minimal. Operator focused. Trustworthy.
Progress bar always visible at the top.

Step 1: Welcome
Headline:
Build Your Financial Profile
Subtext:
Add it once. Use it across every deal.
Two Options:
Upload Existing PFS
Build Profile in Minutes

PATH A — Upload Existing PFS
Upload
Upload full PFS PDF
System Auto-Extraction
System parses and extracts:
Assets
Liabilities
Real Estate
Entities
Income
Review Screen
User lands in review screen
Inline edit enabled
Progress automatically jumps to 70–80%
Action
Confirm and Continue

PATH B — Manual Build Flow
(User can upload documents inside each section. Uploads can automatically route to document storage.)

Section 1: Basic Info
Keep light. One clean block.
Fields:
Full Legal Name
Primary Address
Email
Phone
SSN

Section 2: Liquidity
Single clean layout with + Add Account button.
(Add account must feel seamless and fast.)
Account Card
Institution
Account Type
Balance
Optional Upload:
Bank statement to auto-fill
Cards stack vertically.

Section 3: Real Estate
Headline:
Properties You Own
Single + Add Property button.
Property Card (Minimal Fields)
Address
Property Type
Estimated Value
Loan Balance
Monthly Rent
Optional Upload Inside Card:
Mortgage statement or rent roll to auto-fill
Advanced Dropdown (Optional)
Interest Rate
Monthly Payment
Lender
Maturity Date
Ownership %
Options:
Duplicate property
Cards stack vertically.
No separate screens.

Section 4: Businesses & Entities
+ Add Entity
Entity Card
Entity Name
Ownership %
Estimated Value
Optional Upload:
Operating agreement to auto-fill
Cards stack vertically.

Section 5: Other Assets
Fields:
Investments Total
Private Investments
Other Assets
Optional Upload:
Documents to auto-fill

Section 6: Liabilities
Fields:
Credit Cards Total
Personal Loans
Other Debt
Optional Upload:
Statements to auto-fill
If liability relates to a property or entity:
Allow linking to that asset

Section 7: Income
Fields:
Primary Income
Rental Income
Other Income
Optional Upload:
Tax return to auto-fill

Final Step: Yes / No Questions
Toggle style. Clean layout.
Yes / No questions
If "Yes" → expand small detail box

Final Screen: Snapshot
Display Summary
Total Assets
Total Liabilities
Estimated Net Worth
(Net Worth should appear at the top of the PFS in the actual software.)
Liquidity
Primary CTA
Activate Profile
Upon activation:
Guided navigation through the software
Encourage document uploads
Introduce core features
Help user understand how to use the platform

UX Notes
Minimal copy
Clean vertical stacking
No unnecessary screens
Inline edits wherever possible
Always-visible progress bar
Operator-grade trust aesthetic (bank-level feel)
